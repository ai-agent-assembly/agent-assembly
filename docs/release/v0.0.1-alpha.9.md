## v0.0.1-alpha.9 ‚Äî no new blocker, ships node-sdk content downstream

Ninth pre-release in the v0.0.1 dry-run series. Unlike alpha-5 through
alpha-8, this bump is **not** a recovery from a specific publish-crates
blocker ‚Äî alpha-8 completed end-to-end for all 14 publishable crates +
the homebrew, npm, PyPI, GHCR, and docs channels. alpha-9 carries
agent-assembly docs polish forward and drives a fresh fan-out so SDK
content merged since alpha-8 lands downstream.

### Why bump

The motivating content lives in the sibling SDK repos. Each SDK
`release-*.yml` workflow ships its repo's current master HEAD when the
agent-assembly tag fires `repository_dispatch` ‚Äî so a fresh tag is the
mechanism for getting recently-merged SDK content onto npm / PyPI.

For node-sdk specifically, master HEAD since `@agent-assembly/sdk@0.0.1-alpha.8`
includes:

- **AAASM-2842** ‚Äî public `GatewayClient` + `createNoopGatewayClient` re-exports;
  enables offline tool governance via the public surface.
- **AAASM-2199** ‚Äî README link to `agent-assembly-examples` for runnable samples.
- **AAASM-2829** ‚Äî docs archive backfill for alpha-3 / alpha-4 / alpha-6
  (URL slots restored on the docs site; not part of the npm artifact).

agent-assembly's own master HEAD since alpha-8 adds:

- **AAASM-2199** ‚Äî README link to examples.
- **AAASM-2827** ‚Äî docs archive retention (`extra_archived` seed + rebuild-every-tag-from-git CI).
- **AAASM-2833** ‚Äî dynamic GitHub-release badge alongside crates.io badge.
- **AAASM-2841** ‚Äî version-selector typography polish.

None of this changes any public Rust API, the wire protocol, or crate
contents on crates.io. The Rust workspace is functionally identical to
alpha-8 at the binary level ‚Äî the publish exists primarily to keep
crates.io rows aligned with the SDK channel and to validate that the
release flow is still healthy on a content-stable master.

### Expected post-tag sequence

1. `release.yml` fires ‚Üí builds binaries ‚Üí creates GH Release ‚Üí `publish-crates`
   re-publishes all **14 publishable crates** at `0.0.1-alpha.9` alongside their
   existing `0.0.1-alpha.8` rows.
2. `docker.yml` fires ‚Üí builds + pushes ghcr.io images.
3. `notify-downstream` ‚Üí `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-alpha.9` + 4 sub-packages to npm.
   The docs-version PR auto-opens AND auto-merges via the AAASM-2794 PAT flow.
5. python-sdk publishes `agent-assembly==0.0.1a9` to PyPI.
6. `update-homebrew-tap` opens a tap PR.
7. `release-status-aggregator` posts the per-channel verdict comment.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-alpha.9/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io
cargo install aasm --version 0.0.1-alpha.9

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-alpha.9

# Language SDKs
pip install --pre agent-assembly==0.0.1a9
npm install @agent-assembly/sdk@0.0.1-alpha.9
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-alpha.5  # go-sdk owns its own trajectory
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from alpha-5 / alpha-6 / alpha-7 / alpha-8. The published `aasm`
binary omits the `aasm run <tool>` and `aasm tools` subcommands while the
dev-tool subsystem is being finished. Local source builds (`cargo build -p aa-cli`)
expose the full surface unchanged.
