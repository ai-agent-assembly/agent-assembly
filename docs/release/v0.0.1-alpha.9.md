## v0.0.1-alpha.9 — first coordinated release after the SDK decoupling chapter

Ninth pre-release in the v0.0.1 dry-run series. Validates that the
coordinated `repository_dispatch` fan-out continues to work end-to-end
after the AAASM-2851 SDK release decoupling (which restructured both
node-sdk's `release-node.yml` and python-sdk's `release-python.yml`
+ composite action).

### Why a fresh coordinated bump

alpha-8 completed end-to-end. The motivation for alpha-9 is to:

1. **Regression-validate** that the coordinated release path still
   works after the AAASM-2851 restructure. The SDK decoupling was
   explicitly designed to be back-compat on `repository_dispatch`;
   alpha-9 confirms that on real fan-out.
2. **Ship recent agent-assembly docs polish** (AAASM-2199, 2827,
   2833, 2841, 2858) baked into the alpha-9 docs deploy.
3. **Drive fresh SDK publishes** so `@agent-assembly/sdk@0.0.1-alpha.9`
   and `agent-assembly==0.0.1a9` land on their registries via the
   coordinated path, carrying recent SDK content downstream.

### Expected post-tag sequence

1. `release.yml` → builds binaries → GH Release → `publish-crates`
   re-publishes all 14 publishable crates at `0.0.1-alpha.9`.
2. `docker.yml` → ghcr.io images at the new tag.
3. `notify-downstream` → `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-alpha.9` + 4 runtime
   sub-packages. **Live-validates AAASM-2857's repository_dispatch
   path** with the refactored Resolve step + composite action.
5. python-sdk publishes `agent-assembly==0.0.1a9`.
6. `update-homebrew-tap` opens a tap PR.

### Validation context

This is the **first coordinated release after the AAASM-2851 chain**.
If anything regressed on the `repository_dispatch` path during the
decoupling work, it surfaces here. Pre-release static + dry-run
verification (AAASM-2859, 2862-dispatched validation) suggests no
regression, but coordinated end-to-end is the authoritative test.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-alpha.9/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io
cargo install aasm --version 0.0.1-alpha.9

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-alpha.9

# Language SDKs
pip install --pre agent-assembly==0.0.1a9
npm install @agent-assembly/sdk@0.0.1-alpha.9
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-alpha.5  # go-sdk owns its own trajectory
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from alpha-5 through alpha-8. The published `aasm` binary
omits the `aasm run <tool>` and `aasm tools` subcommands while the
dev-tool subsystem is being finished. Local source builds
(`cargo build -p aa-cli`) expose the full surface unchanged.
