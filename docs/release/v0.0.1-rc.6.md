## v0.0.1-rc.6 — test-quality + tooling hardening

Sixth release candidate in the v0.0.1 series (patch on the `rc` channel,
`rc.5 → rc.6`). A maintenance cut: it clears the outstanding dashboard SonarCloud
findings, guards the eBPF crate's probe build so docs.rs can build it, folds in a
few correctness/authz fixes across the API and CLI surface, and lands
release-process / CI-docs improvements. No new product surface of note.

> **Not for production use.** At `0.x.y` SemVer, rc carries no API/ABI/wire-protocol
> stability commitment. `protocol/v1` is unchanged.

### Correctness & authz fixes

- **Tenant ownership enforced in `register_op`** (`aa-api`) — op registration now
  rejects cross-tenant registration instead of trusting the request's tenant,
  closing a cross-tenant authorization gap. Regression test added.
- **`OpsRegistry::register` no longer clobbers an existing op** (`aa-gateway`) —
  re-registering an existing name preserves the prior op.
- **`aa-cli` audit/logs requests carry the `Authorization` header** — the audit and
  logs subcommands now authenticate against a secured gateway.
- **`/health` alias** (`aa-api`) — adds a `/health` route returning the health JSON,
  for probes expecting that path.

### Dashboard test-quality / SonarCloud (AAASM-4694)

Parameterized the `ApprovalAnalyticsPanel`, `FilterBar`, and `FleetHealthPanel`
tests (S5976), keyed heatmap cells by date instead of index (S6479), named a
`useState` setter symmetrically (S6754), switched base64url decoding to
`replaceAll` (S7781), and used `Set.has` for JWT scope validation (S7776).

### Build / tooling

- **`aa-ebpf` skips its probe subprocess build under `DOCS_RS`** (AAASM-4715) — the
  read-only docs.rs sandbox cannot run the probe build, so it is skipped when
  `DOCS_RS` is set, unbreaking the docs.rs build.

### Release process & CI docs

- 3-part branch-naming convention unified across `CONTRIBUTING.md` (AAASM-4679).
- DCO sign-off checkbox added to the PR template; no-ticket community contribution
  path documented (AAASM-4671 / templates).
- Internal README doc-link check added in CI (AAASM-4674).
- `release-tag-cut` reminded to advance the Jira Fix Version ladder (AAASM-4724).
- `release-docs-sync` extended to cover the README maturity/version string
  (AAASM-4670) — the README is now bumped as part of the release-docs footprint.
