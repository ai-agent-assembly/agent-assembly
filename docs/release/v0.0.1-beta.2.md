## v0.0.1-beta.2 — forward-roll beta carrying the AAASM-3000 IPC deadlock fix

Second pre-release in the v0.0.1 beta channel. This is a **forward-roll**
beta cut on top of `v0.0.1-beta.1` (no channel promotion, no scope
expansion) carrying the AAASM-3000 IPC deadlock fix in `aa-sdk-client`
plus the AAASM-2959 release-tooling sync that keeps the SDK FFI
lockfiles consistent on every coordinated bump.

> **Not for production use.** Beta still carries no API/ABI/wire-protocol
> stability commitment at 0.x.y SemVer.

### agent-assembly content riding this tag

Master content merged since `v0.0.1-beta.1`:

- **AAASM-3000** (Bug) — `aa-sdk-client` IPC event reporting is now
  **fire-and-forget**, closing the deadlock that occurred when the
  runtime accepted but did not ack the event report. Previously, an
  SDK could block on `send_event` against a non-acking runtime; the
  channel is now a one-shot enqueue and `send_event` returns as soon
  as the codec has accepted the frame.
- **AAASM-2959** (Release tooling) — the coordinated-release pipeline
  now syncs `aa-ffi-python` and `aa-ffi-node` `Cargo.lock` when bumping
  the workspace SDK pins, so the published native bindings always
  match the tagged `aa-sdk-client` revision.

### Coordinated SDK publishes driven by this tag

- `@agent-assembly/sdk@0.0.1-beta.2` (npm) — via `repository_dispatch` → `release-node.yml`.
- `agent-assembly==0.0.1b2` (PyPI) — via `repository_dispatch` → `release-python.yml`.
- `github.com/ai-agent-assembly/go-sdk@v0.0.1-beta.2` (Go module) — tagged
  directly on go-sdk (it owns its own release trajectory; not part of the
  agent-assembly fan-out).

### Expected post-tag sequence

1. `release.yml` → builds binaries → GH Release → `publish-crates`
   re-publishes all 14 publishable crates at `0.0.1-beta.2`.
2. `docker.yml` → ghcr.io images at the new tag.
3. `notify-downstream` → `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-beta.2` + 4 runtime sub-packages.
5. python-sdk publishes `agent-assembly==0.0.1b2`.
6. `update-homebrew-tap` opens a tap PR.
7. go-sdk `v0.0.1-beta.2` tag (pushed separately) → goreleaser + Go module proxy.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-beta.2/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io
cargo install aasm --version 0.0.1-beta.2

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-beta.2

# Language SDKs
pip install --pre agent-assembly==0.0.1b2
npm install @agent-assembly/sdk@0.0.1-beta.2
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-beta.2
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from `v0.0.1-beta.1`. The published `aasm` binary omits the
`aasm run <tool>` and `aasm tools` subcommands while the dev-tool
subsystem is being finished. Local source builds (`cargo build -p aa-cli`)
expose the full surface unchanged.
