## v0.0.1-alpha.5 — full pipeline validated

Fifth pre-release in the v0.0.1 dry-run series. Validates the entire
release pipeline end-to-end with **all** of the alpha-4 recovery
fixes baked into master.

### Context — why a fresh bump rather than recovering alpha-4

alpha-4 published successfully to 5 of 6 channels (GH Release,
Homebrew, npm, PyPI, ghcr.io). Only crates.io ended up
partially-published: `aa-core` landed at `0.0.1-alpha.4` from the
first crate in the topological order, but the remaining 8 crates
never published because `cargo workspaces publish` tripped on a
dirty working tree before AAASM-2346's `--allow-dirty` fix.

`gh run rerun --failed` uses the workflow definition that existed at
the time of the original tag push (pre-2346 fix), so re-running could
not pick up the post-merged improvement. Bumping to alpha-5 with a
fresh tag is the cleanest way to validate the full release flow with
every recovery fix in place. On alpha-5, all 9 crates publish to
crates.io including `aa-core` at a fresh `0.0.1-alpha.5` (the
existing alpha-4 row remains on the registry alongside).

### Recovery fixes verified by this tag

- **AAASM-2346 (PR #846)** — `release.yml` now passes `--allow-dirty`
  to `cargo workspaces publish`, surviving the transient working-tree
  dirtiness caused by `.ci/strip-for-publish.sh`.
- **AAASM-2455 (PR #848)** — smoke `pip` pin (superseded by 2457).
- **AAASM-2456 (PR #849)** — new `docs/release/RUNBOOK.md`,
  `scripts/release-readiness.sh` pre-tag gate, and
  `release-status-aggregator` per-channel verdict comment.
- **AAASM-2457 (PR #867)** — smoke matrix restructured: SDK smoke
  jobs dropped from `release.yml` and a `cargo install aasm` channel
  added.

### Companion fixes in SDK repos

- **node-sdk PR #67 (AAASM-2344)** — `package.json` `repository.url`
  lowercased for npm strict-mode.
- **python-sdk PR #74 (AAASM-2345)** — `release-python.yml` Stage
  step bug-fixes (artifact name collision, env var hoist, job order).
- **python-sdk PR #75 (AAASM-2459)** — `pyproject.toml` AND
  `agent_assembly/__init__.py` `__version__` synced together via a
  shared `.github/actions/sync-version/` composite action.
- **python-sdk PR #76 (AAASM-2460)** — deleted broken upstream
  Chisanan232 personal bumper workflows.

### Expected post-tag sequence

1. `release.yml` fires → builds binaries → creates GH Release →
   `publish-crates` succeeds (8 crates published; `aa-core`
   re-publishes at alpha-5 alongside its existing alpha-4).
2. `docker.yml` fires → builds + pushes `ghcr.io` images.
3. `notify-downstream` → `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-alpha.5` (plus its
   4 runtime sub-packages) to npm.
5. python-sdk's `release-python.yml` syncs `pyproject.toml` +
   `__init__.py` to `0.0.1a5` via the composite action, builds wheels,
   publishes to PyPI.
6. Homebrew tap formula bumped via auto-PR (operator merges).
7. `release-status-aggregator` posts a single per-channel verdict
   comment on the GH Release.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-alpha.5/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io — first fully end-to-end-validated workspace publish
cargo install aasm --version 0.0.1-alpha.5

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-alpha.5
docker pull ghcr.io/ai-agent-assembly/python:3.14-slim

# Language SDKs
pip install --pre agent-assembly==0.0.1a5
npm install @agent-assembly/sdk@0.0.1-alpha.5
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-alpha.5
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from alpha-4. The published `aasm` binary omits the
`aasm run <tool>` and `aasm tools` subcommands while the dev-tool
subsystem is being finished. Local source builds
(`cargo build -p aa-cli`) expose the full surface unchanged. See
`docs/src/compatibility.md` for the restoration recipe.
