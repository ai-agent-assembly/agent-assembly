## v0.0.1-beta.1 — alpha → beta channel promotion

First **beta** pre-release in the v0.0.1 series, promoting the channel up
from `0.0.1-alpha.9`. The alpha dry-run series validated the full
coordinated release pipeline end-to-end (crates.io, npm, PyPI, Homebrew,
ghcr.io, and the `repository_dispatch` SDK fan-out); beta.1 is the first
cut on the new channel and re-runs that proven pipeline.

### Why beta

The alpha-1 → alpha-9 series exercised and stabilised every release
channel and the SDK decoupling (AAASM-2851). With that machinery green and
the SDK documentation now first-class (see below), the pre-release channel
moves to **beta** to signal the product is feature-surfacing-complete for
the v0.0.1 line while the API and wire protocol remain unstable.

> **Not for production use.** Beta still carries no API/ABI/wire-protocol
> stability commitment at 0.x.y SemVer.

### Coordinated SDK publishes driven by this tag

- `@agent-assembly/sdk@0.0.1-beta.1` (npm) — via `repository_dispatch` → `release-node.yml`.
- `agent-assembly==0.0.1b1` (PyPI) — via `repository_dispatch` → `release-python.yml`.
- `github.com/ai-agent-assembly/go-sdk@v0.0.1-beta.1` (Go module) — tagged
  directly on go-sdk (it owns its own release trajectory; not part of the
  agent-assembly fan-out).

### Expected post-tag sequence

1. `release.yml` → builds binaries → GH Release → `publish-crates`
   re-publishes all 14 publishable crates at `0.0.1-beta.1`.
2. `docker.yml` → ghcr.io images at the new tag.
3. `notify-downstream` → `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-beta.1` + 4 runtime sub-packages.
5. python-sdk publishes `agent-assembly==0.0.1b1`.
6. `update-homebrew-tap` opens a tap PR.
7. go-sdk `v0.0.1-beta.1` tag (pushed separately) → goreleaser + Go module proxy.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-beta.1/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io
cargo install aasm --version 0.0.1-beta.1

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-beta.1

# Language SDKs
pip install --pre agent-assembly==0.0.1b1
npm install @agent-assembly/sdk@0.0.1-beta.1
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-beta.1
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from the alpha series. The published `aasm` binary omits the
`aasm run <tool>` and `aasm tools` subcommands while the dev-tool
subsystem is being finished. Local source builds (`cargo build -p aa-cli`)
expose the full surface unchanged.
