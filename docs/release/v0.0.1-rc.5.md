## v0.0.1-rc.5 — embedded dashboard + onboarding docs

Fifth release candidate in the v0.0.1 series (patch on the `rc` channel,
`rc.4 → rc.5`). The headline of this cut is **the dashboard now ships inside the
`aa-api` binary**: the built SPA is embedded at compile time, so a locally-served
gateway serves its UI in-process instead of 404-ing the dashboard route — the
regression rc.4 left behind. Folded in are a fail-loud API-key check, a new
mdBook tabs widget with tabbed installation docs, and the Homebrew tap moving to
a generated formula.

> **Not for production use.** At `0.x.y` SemVer, rc carries no API/ABI/wire-protocol
> stability commitment. `protocol/v1` is unchanged.

### Embedded dashboard SPA (the rc.5 driver)

- **Dashboard embedded into `aa-api`** (AAASM-4517) — a `build.rs` + `include_dir!`
  compiles the built dashboard assets into the `aa-api` binary, so a locally-served
  gateway serves its UI in-process. This fixes the rc.4 **dashboard-404** where the
  route had no assets to serve when running locally.

### Fail-loud API-key validation (AAASM-4572)

`aasm` now validates `AASM_API_KEY` before printing the serving banner, so a
missing or invalid key fails loudly up front rather than the CLI claiming to be
serving with no working credential.

### Onboarding docs — tabs widget + tabbed installation

- **mdBook tabs widget** (AAASM-4566) — a reusable tabbed-content widget for the
  docs site.
- **Tabbed installation instructions** (AAASM-4567) — `installation.md` reworked
  onto the tabs widget so per-method install steps sit side by side.
- **Stable installation anchors** (AAASM-4573 / 4574) — per-tab anchors so deep
  links to a specific install method survive re-rendering.

### Also in this window

- **Homebrew tap via `versions.rb` generator** (AAASM-4520) — the tap formula is
  generated rather than hand-maintained.
- Dependency bumps: `rustls`, `tokio-tungstenite`, `uuid`, `open`,
  `EmbarkStudios/cargo-deny-action`, and `softprops/action-gh-release`.
