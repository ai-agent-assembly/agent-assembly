## v0.0.1-alpha.7 — closes the multi-layer publish chain

Seventh pre-release in the v0.0.1 dry-run series. Re-runs the full
release pipeline with the AAASM-2775 strip-for-publish fix baked
into master plus the org-level + repo-level SDK release-workflow
settings flipped on.

### Context — why a fresh bump rather than recovering alpha-6

alpha-6 published 4 of 5 channels (GH Release, Homebrew tap PR #13,
npm `@agent-assembly/sdk@0.0.1-alpha.6` + 4 sub-packages, PyPI
`agent-assembly==0.0.1a6`, Go module proxy `go-sdk@v0.0.1-alpha.5`)
but failed at `publish-crates` on crates.io with a workspace
resolver error:

```
package `aa-integration-tests` depends on `aa-gateway` with feature
`audit-consumer` but `aa-gateway` does not have that feature.
help: available features: default, redis-cache
```

Root cause: AAASM-2388 added `strip-for-publish:begin audit-consumer`
markers around the `audit-consumer` feature in `aa-gateway/Cargo.toml`
but the matching feature forward in `aa-integration-tests/Cargo.toml`
(`audit-consumer = ["aa-gateway/audit-consumer"]`) was never wrapped
and the file was not in MARKED_FILES. `aa-integration-tests` is
`publish = false`, but cargo-workspaces walks the full workspace
graph during publish and resolution fails on the dangling reference.

`gh run rerun --failed` uses the workflow definition at the time of
the original tag push (pre-AAASM-2775 fix), so re-running cannot pick
up the post-merged improvement. Bumping to alpha-7 with a fresh tag
validates the entire release flow end-to-end with the fix in place.

### Recovery fix verified by this tag

- **AAASM-2775 (PR #1021)** — strip-for-publish now also wraps
  `aa-integration-tests/Cargo.toml`'s `audit-consumer` feature
  forward with `strip-for-publish:begin audit-consumer` /
  `:end audit-consumer` markers, and the file is added to
  `MARKED_FILES` in `.ci/strip-for-publish.sh`. The published
  workspace graph no longer references the stripped feature.

### Companion SDK-workflow fixes (no code change required)

The alpha-6 fan-out also surfaced two SDK-release-workflow breakages
that have been resolved via repo / org settings:

- **node-sdk `release-node.yml`** — "Open docs-version PR" step
  failed with `GitHub Actions is not permitted to create or approve
  pull requests`. Cause: org-level Actions setting *Allow GitHub
  Actions to create and approve pull requests* was off; the
  workflow's `pull-requests: write` permission alone wasn't enough.
  Fix: org-level `can_approve_pull_request_reviews` flipped to
  `true` (auto-propagated to all 6 org repos).
- **go-sdk `Docs Site`** — `deploy` job died in 1 second with 0 steps
  on the `v0.0.1-alpha.5` tag push. Cause: the `github-pages`
  environment's deployment-branch-policy allowed only `master`. Fix:
  added a `v*` tag policy alongside the existing `master` policy. The
  rerun succeeded and `ai-agent-assembly.github.io/go-sdk/v0.0.1-alpha.5/`
  is live.

Both fixes apply to all future cycles starting with this alpha-7 tag.

### Expected post-tag sequence

1. `release.yml` fires → builds binaries → creates GH Release →
   `publish-crates` succeeds (3 republished + 6 first-time publishes
   on crates.io — `aa-core`, `aa-proto`, `aa-ebpf-common` re-publish
   at alpha-7 alongside their existing alpha-5 rows; `aa-ebpf`,
   `aa-runtime`, `aa-proxy`, `aa-sandbox`, `aa-gateway`, `aa-cli`
   publish for the first time at alpha-7 alongside their never-
   published alpha-5 / alpha-6 rows).
2. `docker.yml` fires → builds + pushes ghcr.io images.
3. `notify-downstream` → `repository_dispatch` to node-sdk + python-sdk.
4. node-sdk publishes `@agent-assembly/sdk@0.0.1-alpha.7` + 4 sub-
   packages to npm AND opens its own docs-version PR (now permitted).
5. python-sdk publishes `agent-assembly==0.0.1a7` to PyPI.
6. `update-homebrew-tap` opens a tap PR with new shas.
7. `release-status-aggregator` posts a single per-channel verdict
   comment on the GH Release.

### Install

```bash
# Native binaries
brew install ai-agent-assembly/homebrew-agent-assembly/aasm
curl -L https://github.com/ai-agent-assembly/agent-assembly/releases/download/v0.0.1-alpha.7/aasm-aarch64-apple-darwin.tar.gz | tar xz

# crates.io — first end-to-end validated publish of all 9 crates ever
cargo install aasm --version 0.0.1-alpha.7

# Container images
docker pull ghcr.io/ai-agent-assembly/aa-runtime:v0.0.1-alpha.7
docker pull ghcr.io/ai-agent-assembly/python:3.14-slim

# Language SDKs
pip install --pre agent-assembly==0.0.1a7
npm install @agent-assembly/sdk@0.0.1-alpha.7
go get github.com/ai-agent-assembly/go-sdk@v0.0.1-alpha.5  # last go-sdk tag
```

### Behaviour delta on the crates.io `aasm` binary

Unchanged from alpha-5 / alpha-6. The published `aasm` binary omits
the `aasm run <tool>` and `aasm tools` subcommands while the dev-tool
subsystem is being finished. Local source builds
(`cargo build -p aa-cli`) expose the full surface unchanged. See
`docs/src/compatibility.md` for the restoration recipe.
